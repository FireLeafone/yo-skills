# novel-design

此技能用于小说开始前的设计规划：内容大纲、世界人物等的设定、核心主题等等

- 如果未指定文件存放要求，则默认`chapters` 目录下对各个设计单独存放。
- 如果各个设计部分有自己的格式和内容要求，则按文件要求来。
- 尽可能设计完整、逻辑清晰、设定合理
- 头脑风暴式和用户讨论确认设计

## 构建原则
1.  **内部一致性**：规则不能自相矛盾
2.  **服务故事**：世界为故事服务，不要过度构建
3.  **展示而非讲述**：通过情节展现世界
4.  **留有空间**：给读者想象空间
