# novel-write

使用此技能，编写小说章节内容

- 格式：Markdown，文件名和内容位置按要求来，没明确位置，直接放项目根目录。
- 篇幅：每章 不少于 **2000** 字，不多于 **4000** 字。
- 读取大纲和小说设定: 读取 大纲.md和各种设定，锁定目标章节的细纲、登场人物及爽点安排。
- 检查进度: 读取进度文档，确认目标章节是否为"待写"状态，并回顾上一章的结尾以确保剧情衔接。
- 记忆回溯: 如果不是第一章，需检索前文（特别是最近3章）的关键剧情、伏笔和人物状态，防止吃书。
- 内容要求和风格，优先使用用户提供的来，如果没提供，则可以和用户沟通询问这章的情节内容要求（可以根据之前请求和设定，头脑风暴反复和用户确认这章情节接下去的内容和方向）、写作风格等。
- 风格可以供用户选择，内置不少于5种风格，例如（不限于）：幽默风趣、犀利紧凑、叙事陈述、感人煽情、场面宏大等等。
- 多用动作表情描述、对话表现，不要都使用或直接陈述
- 适当添加冲突、悬念、转折等，使情节有活力，
- 逻辑自洽，情节发展必须符合因果逻辑
- 适当情节下添加伏笔，反哺后续情节
- 中断续写：自动检测未完成项目，从断点继续创作，承接上文情节和氛围

## 编写流程 

先规划思考讨论、再编写章节模板、再开始写章节内容、最后写完要做review（检查清单、规范要求等）

## 写作技巧

关键技巧速览：

-   **叙事视角**：第一人称/第三人称限知/全知，根据故事需要选择
-   **场景构建**：时间+地点+人物+冲突+变化
-   **对话写作**：推动情节、展现性格、有潜台词
-   **节奏控制**：张弛有度，高潮后需缓冲
-   **悬念伏笔**：提前埋设、后期回收

起承转合

-   **起**：开篇引入，交代背景和人物
-   **承**：发展情节，层层推进
-   **转**：转折变化，出人意料
-   **合**：收束全文，点明主题

## 文字风格要求

-   **叙述语言**：简洁流畅，避免冗长句式
-   **对话风格**：自然生动，符合人物身份
-   **描写手法**：重点突出，详略得当
-   **修辞运用**：恰到好处，不过度堆砌

## 个人语料与表达指纹

1、口头禅与常用表达

-   示例：我嘴快，但心里有数
-   示例：先把话挑明，再慢慢办
-   示例：这事儿吧，怎么说呢
-   示例：得，就这么定了

2、 固定句式与节奏偏好

-   示例：长句里讲道理，短句落锤
-   示例：段落喜欢先摆事实，后给判断
-   示例：爱用"...但是吧"转折
-   示例：习惯性重复关键词强调

3、 词汇选择偏好

-   **常用词替代**
    -   把"因此"说成"所以"、"这么着"
    -   把"然而"说成"可是"、"但是吧"
    -   把"非常"说成"特别"、"老"、"挺"
    -   把"或许"说成"也许"、"说不定"、"兴许"
-   **量词习惯**
    -   一个人/一位人、一只狗/一条狗
    -   一顿饭/一餐饭、一场雨/一阵雨
-   **形容词口味**
    -   少用"美丽"多用"好看"、"漂亮"
    -   少用"优秀"多用"不错"、"挺好"
    -   避免书面化形容词，偏爱口语化

4、 句子构造习惯

-   **习惯性开头**
    -   那什么、我跟你说、你知道吗
    -   说真的、讲道理、实话实说
    -   要我说、依我看、咱这么想
-   **句中口头禅**
    -   就是说、那个啥、怎么说呢
    -   对吧、是不是、你说呢
    -   这个吧、那啥、反正吧
-   **结尾语气词**
    -   吧、啊、呢、嘛、哈
    -   的、了、着、过（动态助词使用偏好）
    -   省略号使用习惯（...或。。。）

5、行业/地域词汇（口音、俚语、术语）

-   示例：北方口音词：咣当、抻、撂
-   示例：行业术语：回填、预案、解骚（中医）
-   示例：方言词：嘎嘣脆、溜达、唠嗑
-   示例：网络用语的使用频率和类型
