# 执行计划

## 概述

加载计划，进行批判性审查，执行所有任务，完成后报告。

**开始时宣告：** "我正在使用`execute-plans`技能来实施此计划。"

**注意：** 告诉你的合作伙伴，如果能够使用`subagent-development`，效果会更好。如果在支持子代理的平台上运行（如 Claude Code 或 Codex），其工作质量会显著提高。如果子代理可用，请使用 `/yo-dev subagent-development` 替代此技能。

## 流程

### 第一步：加载并审查计划
1. 阅读计划文件
2. 进行批判性审查 — 找出计划中的任何问题或疑虑
3. 如有疑虑：在开始之前向你的合作伙伴提出
4. 如无疑虑：创建 Todo-Write 并继续

### 第二步：执行任务

对于每个任务：
1. 标记为 in_progress（进行中）
2. 严格按照每一步执行（计划中的步骤应简明扼要）
3. 按指定运行验证
4. 标记为 completed（已完成）

### 第三步：完成开发

所有任务完成并验证后：
- 宣告："我正在使用完成开发分支技能来完成此工作。"
- **必需子技能：** 使用 `/yo-dev request-code-review`
- 遵循该技能来验证测试、展示选项、执行选择

## 何时停止并寻求帮助

**立即停止执行的情况：**
- 遇到阻碍（缺少依赖、测试失败、指令不明确）
- 计划存在严重缺陷，无法开始
- 你不理解某个指令
- 验证反复失败

**请求澄清，而不是猜测。**

## 何时回顾前面的步骤

**返回审查（第一步）的情况：**
- 合作伙伴根据你的反馈更新了计划
- 基本方法需要重新思考

**不要强行突破阻碍** — 停下来询问。

## 记住

- 首先批判性地审查计划
- 严格按照计划步骤执行
- 不要跳过验证
- 当计划要求时引用相关技能
- 遇到阻碍时停止，不要猜测
- 未经用户明确同意，切勿在 main/master 分支上开始实施
