## 文案展示

1、按钮文案
	
相同的操作按钮文案全局一致；

- 用动词+名词的形式：「新建表单」「删除文件」；
- 确认类按钮：「确定」「保存」「提交」；
- 取消类按钮：「取消」「关闭」「返回」；
- 危险操作：「删除」需要醒目的视觉提示和二次确认；

2、提示文案
	
- 成功提示：「创建成功」「保存成功」「操作成功」；
- 错误提示：准确描述错误原因，提供解决方案；
- 空状态提示：友好的引导文案，如「暂无数据，请先创建」；
- 加载提示：「加载中...」「处理中...」；
	
3、图标展示
	
- 与内容匹配，符合常规理解；
- 悬浮有文字提示；
- 更多操作，浮框展示

## 数值展示

1、金额
	
- 指定币种，货币符号；
- 单位：按指定币种展示，如需换算需要说明换算单位；
- 小数位：保留2位小数；
- 千位分隔符：每隔三位数加进一个逗号；
- 金额可能小于0，支持有正有负；
- 超大额：考虑使用单位简写 如：¥1.23亿-由需求方定；

2、计费周期
	
周期为正整数：

- 按年：¥xxx/年
- 按月：¥xxx/月
- 按天：¥xxx/天
- 按次：¥xxx/次

3、数值
	
- 小数位：保留2位小数；
- 千位分隔符：每隔三位数加进一个逗号；
- 超大额：考虑使用单位简写 如：1.23亿-由需求方定；

4、百分比
	
- 必须用符号“%”来表示，非小数首位不能是0；
- 保留2位小数；

5、时间
	
- 完整日期：YYYY-MM-DD HH:mm:ss
- 仅日期：YYYY-MM-DD
- 相对时间：「刚刚」「X分钟前」「X小时前」「X天前」
- 时间区间：「2024-03-01 至 2024-03-31」
- 是否精确到时分秒，由需求确定

## 数据录入

1、邮箱地址
	
- 邮箱格式：邮箱正则校验，必须符合邮箱格式；
- 邮箱长度：未避免邮箱长度不足导致不能正常输入，前后端可以设置邮箱长度适当，如100字符等，保证常规格式邮箱都可以正常输入。
	
2、手机号
	
- 手机号长度：移动、联通、电信一般是11位手机号，
- 手机号格式：手机号正则校验，考虑号段可能有更新，可以不做前三位号码合法性校验；
- 考虑员工地域性特点，需要兼容港澳台电话号码格式；

3、固定电话
	
- 固定电话格式：固定电话正则校验；
	
4、身份证
	
- 身份证格式校验；
- 身份证长度：通常为18位、还存在15位；
- 出生日期：如页面有身份证、出生日期两个字段，可以考虑由身份证自动带回出生日期；
- 性别：如页面有身份证、性别两个字段，可以考虑由身份证自动带回性别；

5、金额输入
	
- 设置最大最小值，由需求方定。
	
6、百分比输入
	
- 百分比最大值100，最小值0；
- 小数：最多保留两位小数-由需求方定；

7、日期范围
	
- 跨度限制：需要根据功能确认，防止查询数据量太大；

8、网络相关
	
- IP地址：ipv4，ipv6格式校验；
- 端口范围校验；
- mac地址校验；
- 域名格式校验；
	
9、加密
	
- 加密方式对应格式校验；

## 表单相关

1、必填项
	
- 必填项：使用红色*标识，统一放在label左侧;
- 必填项校验非空（空串也为空）;

2、单选框
	
- 只能选择一个；
- 明确是否有默认值；

3、多选框
	
- 复选框可以都被选中；
- 复选框可以被部分选中；
- 可以逐一选中复选框；

4、输入框
	
- 长度限制：最大长度，需求未明确时，需要根据功能设置最大值，防止数据库字段长度不够导致保存异常；
- 是否必填：如是必填项，则有必填校验和提示；
- 特殊字符限制：由需求方定；
- 首尾去空格：查询、保存数据时，首尾空格去掉；
- 默认最大长度限制，例：名称类：200字；描述类：500字；

5、下拉框
	
- 下拉框选项超过5个或为动态选项，要支持选项模糊搜索；
- 下拉框数据较多时，用接口获取选项；

6、表单校验
	
- 即时验证：输入时/失焦时，同个系统需统一；
- 或提交验证：表单提交时；
- 自定义验证：特定业务场景；
- 校验不通过，需提示错误原因；

7、表单操作
	
- 表单点击“取消”（保存/提交），数据不保存/提交；
- 表单“保存”，保存按钮添加loading，防止多次点击；
- 表单”新建“，不会带回缓存的数据(连续创建的场景易出现此问题)；
- 表单“查看”，不能进行数据保存；

## 页面布局相关

1、表单布局
	
不同功能分类可能表单布局有区别，按照设计师设计规范设置！！
普通情况下：

- 表单尺寸：系统全局一致；
- 标签对齐方式：系统全局一致；
- 标签是否显示冒号：系统全局一致；
- 标签不换行，中英文切换的系统需要注意英文下，label不折行；

2、列表页布局
	
- 筛选重置按钮是否展示、重置是否查询：系统全局一致；
- 筛选区域与页面操作区域分开展示，筛选条件的多少不影响页面布局；
- 筛选区域响应式布局，防止缩小屏幕时筛选区内容重叠；
- 屏幕缩小后，内容可见-列定宽，超出显示滚动条 ；

3、报表、概览页布局
	
- 响应式布局，支持小屏幕-组件库布局中的xs尺寸；
- pc端应用，需要确定最小屏幕尺寸；

4、页面元素布局
	
- 间距：
  - 间距8px基准；
  - 定义间距等统一变量；
  - 页面中同类间距，系统全局一致；
- 圆角：圆角大小全局一致；
- 模块：不同模块视觉区分，避免页面零散的感觉；
- 内容超出：
  - 超出省略时，支持查看全部内容；
  - 超长、超高滚动；
	
5、菜单布局
	
- 左右布局时：缩放-文案隐藏 -由需求方定
	
## 操作交互

**列表模块**

1、表格列
	
- 表头：不折行；
- 列内容：不折行，超出隐藏，悬浮显示全部内容；

2、分页
	
- 每页数据条数与页面列表一致；
- 可以正常分页，支持跳入指定页；
- ES不支持直接跳到某页或页尾；

3、排序
	
- 一般情况按数据记录更新时间倒序（后端）；
- 注意通过定时任务与手动更新数据的更新时间排序；
- 根据实际业务，比如账单/回款列表，通过金额、月份等组合排序-由需求方定；
- 以数据形式呈现字段，均支持正序/倒序排序-由需求方定；

4、数据内容
	
- 如无特殊要求，页面不能出现NULL或NAN等数据；
- 页面首次进入：接口依赖参数需要校验是否为空，为空时不应该调用该接口； 

5、按钮
	
- 删除按钮：删除等危险操作时，需要二次确认；
- 下载、导出按钮：下载、导出中显示loading，防止多次点击；
- 按钮权限：无权限的按钮不显示或置灰-由需求方定；

6、列表导出
	
- 常规情况，导出文件中数据、字段顺序与页面保持一致，并且要导出全部字段；
- 导出时涉及到金额字段，底部自动增加汇总列；
- 支持筛选导出；
- 支持全量导出，请求不超时，服务不重启；

7、状态展示
	
- 需要强调的状态：状态区分，全局统一状态颜色区分；

8、详情页
	
- 刷新：详情页支持刷新后数据正确；
- 操作：详情页区分新建、查看和编辑状态；
- 权限：详情页和进入详情页的按钮权限应该一致；


**筛选模块**

1、无筛选条件
	
- 筛选出全部数据；
- 筛选出空数据；

2、有条件筛选
	
- 筛选出数据与条件选择一致；
- 默认支持模糊/精准搜索；
- 输入框筛选自动去掉前后空格；
- 实时查询加防抖；
	
3、大小写字母
	
- 不区分字母大小写，比如ABC、abc、AbC，都可以搜到ABC、abc的数据；

4、回车自动查询
	
- 选择条件后触发查询、选择条件后用户点击查询按钮，两种情况在一个系统中做到统一；
- 当下拉选项较多时（比如5个），要支持选项的下拉模糊搜索 ；

**权限交互**

1、页面路径（地址栏）
	
- 没权限的页面，入口不可见；
- 没权限的页面，直接输入地址-需要给出友好提示，全局统一：
- 接口报错提示；
  - 或页面数据为空;
  - 或404；
	
2、菜单
	
- 没授权的菜单，不可见；
	
3、按钮
	
- 没权限的按钮，不可见或置灰-由需求方定；
	
4、数据
	
- 没有数据权限的ID，查询页面或接口，返回提示语	

**文件上传和下载**

1、上传
	
- 文件格式、文件个数、文件大小限制：由需求方定；
- 文件列表：上传完成的文件；
- 上传中状态：上传区域置灰，不可再次点击；

2、下载
	
- 列表行下载：
  - 下载中：当前行按钮loading，不可再点击下载；
  - 下载完成：loading消失，可点击；
- 页面上的下载：
  - 下载中：下载区域loading或者进度条展示，不可再点击下载；
  - 下载完成：loading消失或者进度条消失，可再次点击；

**页面元素交互**

1、鼠标悬停效果
	
- 可点击元素悬停时应有明显的视觉反馈-系统全局一致；
- 按钮、链接等元素需设置 hover 状态样式变化，一般hover上之后，颜色变亮；

2、页面加载
	
- 所有接口请求导致页面变化，必须加上loading；
- 使用统一的加载动画-系统全局一致；
- 预留加载占位以防止页面跳动；
	
3、按钮加载
	
- 点击后立即显示loading状态，防止重复提交；

4、弹窗操作
	
- 操作成功：绿色提示，自动消失；
- 操作失败：红色提示，自动消失或需手动关闭；
- 警告信息：黄色提示；
- 位置统一在页面顶部居中；

5、色彩
	
- 系统颜色统一，同类元素颜色一致；
- 操作中和未操作的元素颜色有区分；
- 页面标题、列表表头和内容颜色区分，列表内容悬停行颜色变化；
- 可点击元素：鼠标hover时，颜色变亮；


## 兼容和性能

  - 项目配置全局CSS Reset；
- JavaScript:
  - 浏览器特性检测，兼容代码；
  - 浏览器支持范围设置；

2、容器适配
	
- 媒体查询；
- 响应式栅格布局；

3、大数据量
	
- 列表数据分页；
- 实时数据防抖处理；
- 大文件上传分片-由需求方定；
- 大图片在不影响分辨率的情况下，进行适当的压缩处理；

4、数据判空
	
- 对需要进一步处理的数据，务必进行判空处理；

## 容错处理

1、页面找不到
	
- 404页面；

2、页面无权限
	
- 入口不可见，不能进入；

3、页面错误
	
- 程序错误导致页面白屏，给出友好的错误提示；

4、接口错误
	
- 对接口错误进行全局统一相应和处理；

## 国际化

- 对应语言文案全局一致；
- 页面布局适应不同语言，不能出现切换语言后，页面布局混乱的情况；
