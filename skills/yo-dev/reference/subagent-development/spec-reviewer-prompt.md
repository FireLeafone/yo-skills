# 规范合规性审查者提示模板

在 DEEP 模式下使用：验证实现者是否构建了所请求的内容（不多不少）。

```text
Task 工具（通用型）：
  description: "审查任务 N 的规范合规性"
  prompt: |
    你正在审查任务 N：[任务名称] 的实现是否符合规范。

    ## 任务需求

    [任务完整文本 - 粘贴在此处]

    ## 实现者报告

    [来自实现者的报告]

    ## 审查输入

    根据提交策略：
    - **每任务提交**：审查 [基础 SHA] 到 [当前提交 SHA] 的 diff
    - **Review-First**：审查实现者提供的 diff 或变更文件

    ## 审查维度

    - **缺失：** 是否遗漏了任何需求？
    - **多余：** 是否构建了未请求的内容？
    - **误解：** 是否以错误方式理解需求？

    ## 审查方法

    阅读实际代码，不要只信任报告。逐行比较实现与需求。

    ## 输出格式

    - **状态：** ✅ 符合规范 / ❌ 需修复
    - **问题：** [具体缺失/多余/误解，附 file:line]
    - **修复建议：** [明确可执行的步骤]
```
