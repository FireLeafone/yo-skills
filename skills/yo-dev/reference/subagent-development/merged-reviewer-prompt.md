# 合并审查者提示模板

在 STANDARD 模式下使用：一次审查同时验证「是否按规范实现」和「实现质量是否合格」。

```text
Task 工具（通用型）：
  description: "审查任务 N：[任务名称]"
  prompt: |
    你正在审查任务 N：[任务名称] 的实现。

    ## 任务需求

    [任务完整文本 - 粘贴在此处]

    ## 实现者报告

    [来自实现者的报告，包括状态、实现内容、测试结果、变更文件]

    ## 审查方式

    根据提交策略选择审查输入：
    - **每任务提交**：比较 [基础 SHA] 到 [当前提交 SHA] 的 diff
    - **Review-First**：审查实现者提供的 working tree diff 或关键 diff

    ## 审查维度

    **1. 规范合规性**
    - 是否实现了所有需求？
    - 是否有多做或少做的内容？
    - 是否误解了需求？

    **2. 代码质量**
    - 代码是否整洁、可维护？
    - 命名是否清晰？
    - 是否有明显重复、魔法数字、过度复杂？
    - 测试是否真实验证了行为？
    - 文件职责是否清晰？

    ## 审查方法

    不要只信任实现者的报告。阅读实际代码/diff，独立验证。

    ## 输出格式

    - **状态：** ✅ 通过 / ❌ 需修复
    - **规范问题：** [具体缺失/多余/误解，附 file:line]
    - **质量问题：** [关键/重要/次要问题，附 file:line]
    - **修复建议：** [明确可执行的修复步骤]
```
