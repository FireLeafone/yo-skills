## 登记

品牌：动效是品牌声音的一部分；一个精心编排的入场动画胜过零散的微交互。饱和的 AI 默认值是每个滚动区域都淡入浮出；那是一个信号，不是编排。只为值得的时刻保留滚动触发的动效。

产品：大多数过渡 150–250ms。动效传达状态：反馈、展示、加载、视图间过渡。不要有页面加载编排；用户正在执行任务，不会等它。

## 评估动画机会

分析动效可以提升体验的地方：

1. **识别静态区域**：
   - **缺少反馈**：操作没有视觉确认（按钮点击、表单提交等）
   - **突兀的过渡**：状态变化过于生硬（显示/隐藏、页面加载、路由切换）
   - **关系不清晰**：空间或层级关系不明显
   - **缺少愉悦感**：功能完整但缺乏趣味的交互
   - **缺少引导**：引导注意力或解释行为的机会

2. **理解语境**：
   - 个性是什么？（活泼 vs 严肃，充满活力 vs 沉稳）
   - 性能预算是多少？（移动优先？复杂页面？）
   - 受众是谁？（对动效敏感的用户？想要速度的高级用户？）
   - 最重要的是什么？（一个重点动画 vs 多个微交互？）

如果代码库中有任何不清楚的地方，询问澄清。

**关键**：尊重 `prefers-reduced-motion`。始终为需要的用户提供非动画替代方案。

## 规划动画策略

创建有目的的动画计划：

- **重点时刻**：那一个标志性动画是什么？（页面加载？首屏区域？关键交互？）
- **反馈层**：哪些交互需要确认？
- **过渡层**：哪些状态变化需要平滑？
- **愉悦层**：哪里可以带来惊喜和愉悦？

**重要**：一个编排良好的体验胜过到处散落动画。聚焦高影响力时刻。

## 实现动画

系统地添加动效，分为以下类别：

### 入场动画
- **首屏区域**：主要内容的戏剧性入场（缩放、视差或创意效果）
- **模态框/抽屉进入**：平滑滑入 + 淡入，背景淡入，焦点管理
- **列表节奏**：同级交错对网格中的卡片或出现的列表项是合理的。整段区域的滚动淡入不是列表，不合理。交错总时长上限：10 项每项 50ms = 500ms 总计。更多项目时，减少每项延迟或限制交错数量。

  使用 CSS 自定义属性实现干净的交错：`animation-delay: calc(var(--i, 0) * 50ms)`，每项设置 `style="--i: 0"`、`style="--i: 1"` 等。

### 微交互

- **按钮反馈**：
  - 悬停：轻微缩放（1.02–1.05）、颜色偏移、阴影增强
  - 点击：快速缩小再放大（0.95 → 1），涟漪效果
  - 加载：旋转或脉冲状态
- **表单交互**：
  - 输入框聚焦：边框颜色过渡、轻微缩放或发光
  - 验证：错误时抖动、成功时勾选、平滑颜色过渡
- **切换开关**：平滑滑动 + 颜色过渡（200–300ms）
- **复选框/单选**：勾选动画、涟漪效果
- **点赞/收藏**：缩放 + 旋转、粒子效果、颜色过渡

### 状态过渡

- **显示/隐藏**：淡入 + 滑动（不要瞬时），适当的时间（200–300ms）
- **展开/折叠**：高度过渡配合 overflow 处理、图标旋转
- **加载状态**：骨架屏淡入、旋转动画、进度条
- **成功/错误**：颜色过渡、图标动画、轻微缩放脉冲
- **启用/禁用**：不透明度过渡、光标变化

### 导航与流程

- **页面过渡**：路由间交叉淡入、共享元素过渡
- **标签切换**：滑动指示器、内容淡入/滑动
- **轮播/滑块**：平滑变换、吸附点、动量
- **滚动效果**：视差层、带状态变化的粘性头部、滚动进度指示器

### 反馈与引导

- **悬停提示**：工具提示淡入、光标变化、元素高亮
- **拖放**：抬起效果（阴影 + 缩放）、放置区域高亮、平滑重新定位
- **复制/粘贴**：粘贴时短暂高亮闪烁、"已复制"确认
- **焦点流程**：通过表单或工作流高亮路径

### 愉悦时刻

- **空状态**：插画的轻微浮动动画
- **完成操作**：五彩纸屑、勾选 flourish、成功庆祝
- **彩蛋**：供发现的隐藏交互
- **情境动画**：天气效果、昼夜主题、季节触感

## 技术实现

为每个动画使用适当的技术：

### 时间与缓动

**时长：100/300/500 规则。** 时间比缓动更能决定"感觉对了"：

| 时长 | 用例 | 示例 |
|----------|----------|----------|
| **100–150ms** | 即时反馈 | 按钮按压、切换、颜色变化 |
| **200–300ms** | 状态变化 | 菜单打开、工具提示、悬停状态 |
| **300–500ms** | 布局变化 | 折叠面板、模态框、抽屉 |
| **500–800ms** | 入场动画 | 页面加载、首屏展示 |

**缓动曲线（使用这些，而非 CSS 默认值）：**
```css
/* 推荐：自然减速 */
--ease-out-quart: cubic-bezier(0.25, 1, 0.5, 1);    /* 平滑 */
--ease-out-quint: cubic-bezier(0.22, 1, 0.36, 1);   /* 稍微干脆 */
--ease-out-expo: cubic-bezier(0.16, 1, 0.3, 1);     /* 自信、果断 */

/* 避免：感觉过时和廉价 */
/* bounce: cubic-bezier(0.34, 1.56, 0.64, 1); */
/* elastic: cubic-bezier(0.68, -0.6, 0.32, 1.6); */
```

**退出动画比入场更快。** 使用入场时长的约 75%。

### CSS 动画
```css
/* 简单、声明式动画优先 */
- transitions 用于状态变化
- @keyframes 用于复杂序列
- transform 和 opacity 用于可靠的移动
- blur、filters、masks、clip paths、shadows 和 color shifts 用于高级氛围效果，在确认流畅后使用
```

### JavaScript 动画
```javascript
/* 复杂、交互式动画使用 */
- Web Animations API 用于程序化控制
- Framer Motion 用于 React
- GSAP 用于复杂序列
```

### 动效材质

Transform 和 opacity 是可靠的默认选择，但不是全部调色板。高级界面通常需要氛围属性。材质与效果匹配：

- **Transform / opacity**：移动、按压反馈、简单展示、列表编排
- **Blur / filter / backdrop-filter**：焦点拉动、景深、玻璃或镜头效果、柔化入场
- **Clip-path / masks**：擦除、展示、编辑裁剪、产品化过渡
- **Shadow / glow / color filters**：能量、可供性、焦点、温暖、激活状态
- **Grid-template-rows 或 FLIP 式变换**：展开和重排布局而不直接动画化 `height`

硬规则不是"只用 transform 和 opacity"。而是：避免随意动画化布局驱动属性（`width`、`height`、`top`、`left`、margin），将昂贵效果限制在小范围或隔离区域，并在目标视口的浏览器中验证流畅度。

### 性能
- **布局安全**：避免随意动画化布局驱动属性（`width`、`height`、`top`、`left`、margin）
- **will-change**：仅对已知昂贵的动画少量添加（例如在 `:hover` 或 `.animating` 类上），永远不要在整页上抢先添加
- **滚动触发**：使用 Intersection Observer 替代滚动事件监听；动画触发一次后取消观察
- **限制昂贵效果**：将 blur/filter/shadow 区域保持小或隔离，适当使用 `contain`
- **监控 FPS**：确保目标设备上 60fps

### 感知性能

没人关心你的网站*有多快*，只关心它*感觉有多快*。80ms 阈值：任何低于约 80ms 的交互都感觉即时，因为我们的大脑会缓冲这么长时间的感官输入来同步感知。以此为目标处理微交互。

- **抢先开始**：加载时立即开始过渡（iOS 应用缩放、骨架 UI）。用户感知到工作正在进行。
- **提前完成**：渐进式展示内容，不要等所有内容（渐进式图片、流式 HTML、骨架淡入）。
- **乐观 UI**：立即更新界面，优雅地处理失败。用于低风险操作（点赞、关注）。避免用于支付或破坏性操作。
- **缓动影响感知时长**：Ease-in（加速到完成）让任务感觉更短，因为峰终效应严重加权最后时刻。Ease-out 让入场感觉令人满意。
- **注意**：过快的响应会降低复杂操作的感知价值（搜索、分析）。有时短暂的延迟表示"真正的工作"正在进行。

### 无障碍
```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

**绝不**：
- 使用弹跳或弹性缓动曲线；它们感觉过时并把注意力引向动画本身
- 随意动画化布局属性（`width`、`height`、`top`、`left`、margin），当 transform、FLIP 或基于 grid 的技术可行时
- 反馈动画超过 500ms（感觉卡顿）
- 无目的地动画化（每个动画都需要理由）
- 忽略 `prefers-reduced-motion`（这是无障碍违规）
- 动画化所有内容（动画疲劳让界面感觉累人）
- 除非有意，否则在动画期间阻止交互

## 验证质量

彻底测试动画：

- **目标设备上 60fps 流畅**：无卡顿
- **感觉自然**：缓动曲线感觉有机，不机械
- **时间适当**：不太快（突兀）也不太慢（卡顿）
- **减少动效有效**：动画适当禁用或简化
- **不阻塞**：用户可以在动画期间/之后交互
- **增加价值**：让界面更清晰或更愉悦

当动效澄清状态而非装饰状态时，交给 `/yo-ui polish` 做最终润色。
