# 基于角色的设计测试

通过 5 个不同的用户原型视角测试界面。每个角色暴露出单一"设计总监"视角会遗漏的不同失败模式。

**使用方法**：选择与被评审界面最相关的 2–3 个角色。以每个角色的主要用户操作。报告具体的红旗，不是通用的担忧。

---

## 角色档案

### 1. 不耐烦的高级用户："Alex"

**简介**：类似产品的专家。期待效率，讨厌手把手。会找到捷径或离开。

**行为**：
- 跳过所有引导和说明
- 立即寻找键盘快捷键
- 尝试批量选择、批量编辑和自动化
- 对感觉不必要的必需步骤感到沮丧
- 如果任何感觉慢或居高临下就放弃

**测试问题**：
- Alex 能否在 60 秒内完成核心任务？
- 常见操作有键盘快捷键吗？
- 引导可以完全跳过吗？
- 弹窗有键盘关闭（Esc）吗？
- 有"高级用户"路径（快捷键、批量操作）吗？

**红旗**（具体报告这些）：
- 强制教程或不可跳过的引导
- 主要操作没有键盘导航
- 无法跳过的慢动画
- 批量更自然的地方使用逐项工作流
- 低风险操作的冗余确认步骤

---

### 2. 困惑的新手："Jordan"

**简介**：从未使用过这类产品。每一步都需要引导。宁放弃也不摸索。

**行为**：
- 仔细阅读所有说明
- 点击任何不熟悉的东西前犹豫
- 不断寻找帮助或支持
- 误解行话和缩写
- 对任何标签做最字面的解释

**测试问题**：
- 第一个操作在 5 秒内是否明显清晰？
- 所有图标都有文字标签吗？
- 决策点有语境帮助吗？
- 术语假设了先验知识吗？
- 每一步都有清晰的"返回"或"撤销"吗？

**红旗**（具体报告这些）：
- 没有标签的仅图标导航
- 没有解释的技术术语
- 没有可见帮助选项或引导
- 完成操作后下一步不明确
- 没有操作成功的确认

---

### 3. 依赖无障碍的用户："Sam"

**简介**：使用屏幕阅读器（VoiceOver/NVDA），仅键盘导航。可能有低视力、运动障碍或认知差异。

**行为**：
- 线性 Tab 浏览界面
- 依赖 ARIA 标签和标题结构
- 无法看到悬停状态或纯视觉指示器
- 需要足够的颜色对比度（最小 4.5:1）
- 可能使用高达 200% 的浏览器缩放

**测试问题**：
- 主要流程能否仅通过键盘完成？
- 所有交互元素是否可聚焦并有可见的焦点指示器？
- 图片有有意义的 alt 文本吗？
- 颜色对比度是否符合 WCAG AA（文本 4.5:1）？
- 屏幕阅读器是否播报状态变化（加载、成功、错误）？

**红旗**（具体报告这些）：
- 只有点击交互没有键盘替代方案
- 焦点指示器缺失或不可见
- 仅通过颜色传达含义（红色 = 错误，绿色 = 成功）
- 未标记的表单字段或按钮
- 没有延长选项的限时操作
- 破坏屏幕阅读器流程的自定义组件

---

### 4. 刻意的压力测试者："Riley"

**简介**：有条理的用户，将界面推到快乐路径之外。测试边界情况，尝试意外输入，探测体验中的缺口。

**行为**：
- 有意测试边界情况（空状态、长字符串、特殊字符）
- 用意外数据提交表单（emoji、RTL 文本、超长值）
- 通过向后导航、流程中途刷新或在多个标签页打开来尝试破坏工作流
- 寻找 UI 承诺与实际发生之间的不一致
- 有条理地记录问题

**测试问题**：
- 边界处发生什么（0 项、1000 项、超长文本）？
- 错误状态是优雅恢复还是让 UI 处于损坏状态？
- 流程中途刷新会发生什么？状态是否保留？
- 有看起来工作但产生损坏结果的功能吗？
- UI 如何处理意外输入（emoji、特殊字符、从 Excel 粘贴）？

**红旗**（具体报告这些）：
- 看起来工作但静默失败或产生错误结果的功能
- 暴露技术细节或让 UI 处于损坏状态的错误处理
- 没有有用信息的空状态（"无结果"没有引导）
- 刷新或导航时丢失用户数据的工作流
- UI 不同部分相似交互之间的不一致行为

---

### 5. 分心的移动用户："Casey"

**简介**：单手在路上使用手机。频繁被打断。可能在慢速连接上。

**行为**：
- 只用拇指；偏好屏幕底部操作
- 流程中被打断，稍后返回
- 频繁切换应用
- 注意力有限，耐心低
- 尽量少打字，偏好点击和选择

**测试问题**：
- 主要操作在拇指区域（屏幕下半部分）吗？
- 如果用户离开并返回，状态是否保留？
- 在慢速连接（3G）上工作吗？
- 表单能使用自动完成和智能默认值吗？
- 触摸目标至少 44×44pt 吗？

**红旗**（具体报告这些）：
- 重要操作位于屏幕顶部（拇指够不到）
- 没有状态持久性；标签切换或打断时丢失进度
- 可以用选择代替的地方需要大量文本输入
- 每页加载大量资源（没有延迟加载）
- 触摸目标太小或彼此太近

---

## 选择角色

根据界面类型选择角色：

| 界面类型 | 主要角色 | 原因 |
|----------|----------|------|
| 落地页 / 营销 | Jordan, Riley, Casey | 第一印象、信任、移动端 |
| 仪表板 / 管理 | Alex, Sam | 高级用户、无障碍 |
| 电商 / 结账 | Casey, Riley, Jordan | 移动端、边界情况、清晰度 |
| 引导流程 | Jordan, Casey | 困惑、打断 |
| 数据密集型 / 分析 | Alex, Sam | 效率、键盘导航 |
| 表单密集型 / 向导 | Jordan, Sam, Casey | 清晰度、无障碍、移动端 |

---

## 项目特定角色

仅在有指定的特定角色及描述时使用。没有时使用 5 个预定义角色。
