# 认知负荷评估

认知负荷是使用界面所需的总脑力。超负荷的用户会犯错、感到沮丧并离开。

## 三种认知负荷

### 内在负荷：任务本身

用户试图完成的事情固有的复杂度。你无法消除这个，但可以结构化。

**通过以下方式管理**：
- 将复杂任务分解为离散的步骤
- 提供脚手架（模板、默认值、示例）
- 渐进式披露：展示现在需要的，隐藏其余
- 将相关决策分组在一起

### 外在负荷：糟糕的设计

由糟糕的设计选择造成的心理努力。**无情地消除这个。** 它是纯粹的浪费。

**常见来源**：
- 需要心理映射的混乱导航
- 强迫用户猜测含义的不清晰标签
- 争夺注意力的视觉杂乱
- 阻止学习的不一致模式
- 用户意图和结果之间不必要的步骤

### 相关负荷：学习努力

用于建立理解的心理努力。这是*好的*认知负荷；它通向精通。

**通过以下方式支持**：
- 逐步揭示复杂性的渐进式披露
- 奖励学习的一致模式
- 确认正确理解的反馈
- 通过行动而非文字墙进行教学的引导

## 认知负荷清单

根据这 8 项评估界面：

- [ ] **单一焦点**：用户能否在没有竞争元素干扰的情况下完成主要任务？
- [ ] **分块**：信息是否以可消化的分组呈现（每组 ≤4 项）？
- [ ] **分组**：相关项目是否在视觉上分组（邻近、边框、共享背景）？
- [ ] **视觉层级**：屏幕上最重要的东西是否立即清晰？
- [ ] **一次一件事**：用户能否在转移到下一个之前专注于单一决策？
- [ ] **最少选择**：决策是否简化（任何决策点可见选项 ≤4）？
- [ ] **工作记忆**：用户是否需要记住前一屏的信息才能在当前屏上行动？
- [ ] **渐进式披露**：复杂性是否只在用户需要时展示？

**评分**：统计失败项。0–1 失败 = 低认知负荷（好）。2–3 = 中等（尽快处理）。4+ = 高认知负荷（需要关键修复）。

## 工作记忆规则

**人类一次最多能在工作记忆中持有 ≤4 项**（Miller 法则由 Cowan 修订，2001）。

在任何决策点，计算用户必须同时考虑的不同选项、操作或信息片段数量：
- **≤4 项**：在工作记忆限制内，可管理
- **5–7 项**：推边界；考虑分组或渐进式披露
- **8+ 项**：超负荷；用户会跳过、误点或放弃

**实际应用**：
- 导航菜单：≤5 个顶级项（其余归入清晰的分类）
- 表单部分：视觉分隔前每组 ≤4 个可见字段
- 操作按钮：1 个主要，1–2 个次要，其余放入菜单
- 仪表板小部件：不滚动可见 ≤4 个关键指标
- 定价层级：≤3 个选项（更多导致分析瘫痪）

## 常见认知负荷违规

### 1. 选项墙
**问题**：一次呈现 10+ 个选择，没有层级。
**修复**：归入分类，高亮推荐，使用渐进式披露。

### 2. 记忆桥
**问题**：用户必须记住第 1 步的信息才能完成第 3 步。
**修复**：保持相关语境可见，或在需要时重复它。

### 3. 隐藏导航
**问题**：用户必须构建事物在哪里的心理地图。
**修复**：始终显示当前位置（面包屑、激活状态、进度指示器）。

### 4. 术语壁垒
**问题**：技术或领域语言强制翻译努力。
**修复**：使用简单语言。如果领域术语不可避免，内联定义它们。

### 5. 视觉噪音地板
**问题**：每个元素都有相同的视觉权重；没有突出的东西。
**修复**：建立清晰的层级：一个主要元素，2–3 个次要，其余弱化。

### 6. 不一致模式
**问题**：相似操作在不同地方工作方式不同。
**修复**：标准化交互模式。同类型操作 = 同类型 UI。

### 7. 多任务需求
**问题**：界面需要同时处理多个输入（阅读 + 决策 + 导航）。
**修复**：排序步骤。让用户一次做一件事。

### 8. 语境切换
**问题**：用户必须在屏幕/标签页/弹窗之间跳转来收集单一决策所需的信息。
**修复**：将每个决策所需的信息放在一起。减少来回。
