空间是最被低估的设计工具。找到布局的真正问题（单调间距、弱层级、千篇一律的卡片网格）并修复结构，而非表面。

---

## 登记

品牌：非对称构图，使用 `clamp()` 的流体间距，为强调而故意打破网格。通过对比创造节奏：紧密分组搭配宽松分隔。

产品：可预测的网格，一致的密度，熟悉的导航模式。响应式行为是结构性的（折叠侧边栏、响应式表格），不是流体排版。一致性本身就是一种可供性。

## 评估当前布局

分析当前空间设计的薄弱之处：

1. **间距**：
   - 间距是否一致还是随意？（随机的 padding/margin 值）
   - 所有间距都一样吗？（到处都是相等的 padding = 没有节奏）
   - 相关元素是否紧密分组，组之间有宽松的间距？

2. **视觉层级**：
   - 应用眯眼测试：模糊你（比喻性的）眼睛。你仍然能识别最重要的元素、第二重要的和清晰的分组吗？
   - 层级是否有效实现？（空间和字重单独可能就足够了；当前方法有效吗？）
   - 留白是否将视线引导到重要的东西上？

3. **网格与结构**：
   - 是否有清晰的底层结构，还是布局感觉随机？
   - 是否到处使用相同的卡片网格？（图标 + 标题 + 文本，无休止地重复）

4. **节奏与变化**：
   - 布局是否有视觉节奏？（交替的紧密/宽松间距）
   - 每个部分的结构都一样吗？（单调重复）
   - 是否有故意的惊喜或强调时刻？

5. **密度**：
   - 布局是否太拥挤？（没有足够的呼吸空间）
   - 布局是否太稀疏？（没有目的的过度留白）
   - 密度是否匹配内容类型？（数据密集型 UI 需要更紧的间距；营销页面需要更多留白）

**关键**：布局问题通常是界面感觉"不对"的根本原因，即使颜色和字体没问题。空间是一种设计材料；有意图地使用它。

## 规划布局改进

创建系统化的计划：

- **间距系统**：使用一致的刻度（框架的内置刻度如 Tailwind 的、基于 rem 的 token 或自定义系统）。具体值不如一致性重要。
- **层级策略**：空间将如何传达重要性？
- **布局方法**：什么结构适合内容？Flex 用于 1D，Grid 用于 2D，命名区域用于复杂页面布局。
- **节奏**：哪里应该紧密，哪里应该宽松？

## 系统化改进布局

### 建立间距系统

- 使用一致的间距刻度（框架刻度如 Tailwind、基于 rem 的 token 或自定义刻度都有效）。重要的是值来自定义集合，不是随意数字。
- 优先使用 4pt 基准刻度（4、8、12、16、24、32、48、64、96px）而非 8pt；8pt 太粗，你经常需要在 8 和 16 之间用 12px。
- 如果使用自定义属性，语义化命名 token：`--space-xs` 到 `--space-xl`，不是 `--spacing-8`
- 使用 `gap` 处理兄弟间距而非 margin；消除 margin collapse 技巧
- 对在大屏幕上呼吸的流体间距应用 `clamp()`

### 创造视觉节奏

- **紧密分组**用于相关元素（兄弟间 8-12px）
- **宽松分隔**用于不同部分（48-96px）
- **部分内部变化间距**（不是每行都需要相同的 gap）
- **非对称构图**：当内容邀请时的有意选择（不是追逐的默认）。

### 选择正确的布局工具

- **Flexbox 用于 1D 布局**：项目行、导航栏、按钮组、卡片内容、大多数组件内部。
- **Grid 用于 2D 布局**：页面级结构、仪表板、数据密集型界面、任何需要行列协调控制的地方。
- 对复杂页面布局使用命名网格区域（`grid-template-areas`）；在断点处重新定义。
- 对组件使用**容器查询**，对视口使用视口查询。窄侧边栏中的卡片可以保持紧凑，而主内容区中相同的卡片自动展开：

```css
.card-container { container-type: inline-size; }
.card { display: grid; gap: var(--space-md); }
@container (min-width: 400px) {
  .card { grid-template-columns: 120px 1fr; }
}
```

### 打破卡片网格单调

- 不要默认对所有内容使用卡片网格；间距和对齐自然创造视觉分组
- 仅在内容真正不同且可操作时卡片化。永远不要在卡片内嵌套卡片
- 变化卡片尺寸、跨列，或将卡片与非卡片内容混合以打破重复

### 强化视觉层级

- 使用清晰层级所需的最少维度。空间单独可能就足够；元素周围的慷慨留白吸引视线。一些最精致的设计只用空间和字重就实现了节奏。仅在更简单的手段不够时才添加颜色或尺寸对比。
- 最佳层级一次结合 2–3 个维度。一个更大、更粗、上方有更多空间的标题无需刻意就能读出是主要的：

| 工具 | 强层级 | 弱层级 |
|------|------------------|----------------|
| **尺寸** | 3:1 比例或更高 | <2:1 比例 |
| **字重** | 粗体 vs 常规 | 中等 vs 常规 |
| **颜色** | 高对比 | 相似色调 |
| **位置** | 顶部/左侧（主要） | 底部/右侧 |
| **空间** | 被留白环绕 | 拥挤 |

- 注意阅读流：在 LTR 语言中，眼睛自然从左上扫描到右下，但主要操作位置取决于语境（例如弹窗中在右下，导航中在顶部）。
- 通过邻近和分隔创建清晰的内容分组。

### 管理深度与海拔

- 构建一致的阴影刻度（sm → md → lg → xl）；阴影应该微妙
- 使用海拔强化层级，不是作为装饰

### 光学调整

- 如果图标尽管几何居中但视觉上看起来不居中，微调它。但仅在你确信它确实看起来错了。不要投机性调整。
- `margin-left: 0` 的文本因字形留白看起来略微缩进；负 margin（`-0.05em`）在光学上对齐它。几何居中的字形通常看起来不居中（播放图标需要向右移，箭头朝向它们的指向移）。
- 即使视觉元素更小，触摸目标也必须最小 44×44px。用 padding 或伪元素扩展点击区域：

```css
.icon-button { width: 24px; height: 24px; position: relative; }
.icon-button::before {
  content: ''; position: absolute; inset: -10px;
}
```

**绝不**：
- 在刻度外使用任意间距值
- 让所有间距相等（变化创造层级）
- 把所有东西都包在卡片里（不是所有东西都需要容器）
- 在卡片内嵌套卡片（内部用间距和分隔线处理层级）
- 到处使用相同的卡片网格（图标 + 标题 + 文本，重复）
- 默认将首屏指标布局（大数字、小标签、统计、渐变）作为模板。如果展示真实用户数据，突出的指标可以工作，但它应该显示实际数据，不是装饰数字。

## 验证布局改进

- **眯眼测试**：你能用模糊视觉识别主要、次要和分组吗？
- **节奏**：页面是否有令人满意的紧密和宽松间距的节奏？
- **层级**：最重要的内容在 2 秒内是否明显？
- **呼吸空间**：布局是否感觉舒适，不拥挤也不浪费？
- **一致性**：间距系统是否统一应用？
- **响应式**：布局是否优雅地跨屏幕尺寸适配？

当节奏和层级到位时，交给 `/yo-ui polish` 做最终润色。

## 实时模式签名参数

每个变体必须声明一个 `density` 参数。通过 `calc(var(--p-density, 1) * <base>)` 驱动变体作用域 CSS 中的所有间距 token：padding、gap、列宽。用户从宽松滑动到紧凑，布局重新呼吸而无需重新生成。

```json
{"id":"density","kind":"range","min":0.6,"max":1.4,"step":0.05,"default":1,"label":"Density"}
```

对于拓扑真正变化的变体（堆叠 vs 并排、网格 vs bento），使用一个 `steps` 参数，其作用域 CSS 通过 `:scope[data-p-structure="X"]` 分支。一个结构参数 + 一个密度参数是强大的组合；抵制添加第三个。

```json
{"id":"structure","kind":"steps","default":"grid","label":"Structure","options":[
  {"value":"stacked","label":"Stacked"},
  {"value":"grid","label":"Grid"},
  {"value":"bento","label":"Bento"}
]}
```
