# novel-chapters

在已有故事大纲和人物设定的基础上，将小说按卷（Volume）拆分，为每卷规划详细的章节安排，并生成总览文档方便追踪整体进度。

## 前置条件

使用本 skill 前，建议项目已具备：
- 故事大纲（`presets/outline.md`）
- 人物设定（`presets/characters.md`）

如果这些文件存在，先快速浏览以了解故事框架和人物关系。如果不存在，提醒用户可先用 `novel-design` 完成前期设计，但仍可按用户提供的信息直接规划。

## 输出位置

- 默认输出到项目根目录下的 `chapters/` 目录。
- **每卷**：`chapters/volume-<N>/index.md`
- **总览**：`chapters/index.md`（卷概况总览）

如果目标目录已有同名文件，**必须先询问用户是覆盖、合并还是跳过**，禁止直接覆盖。

## 工作流程

### 1. 收集信息

在开始规划前，向用户确认以下信息：

**必问项：**
- 总共分为几卷？
- 每卷大概多少章？（默认范围 40–150，按用户指定来）
- 是否有已有的大纲或人物设定可供参考？

**选问项（如果用户在对话中已提供则跳过）：**
- 全书总字数目标？
- 每章目标字数？
- 是否有特定的分卷逻辑（如按时间线、按地图切换、按视角切换）？
- 是否有特殊要求（如某卷节奏更快、某卷需要更多铺垫等）？

信息收集示例：

```markdown
在开始章节规划之前，请确认以下信息：

1. **全书分为几卷？** 比如 3 卷、5 卷、8 卷等。
2. **每卷章节数？** 建议 40–150 章/卷，可以每卷不同，也可以统一。
3. **有没有已有的大纲或人物设定？** 如果有，我会参考它们来规划。
4. **其他特殊要求？** 比如某卷重点、节奏偏好等。
```

### 2. 确认规划方案（硬性门槛）

收集信息后，向用户展示一个规划方案，包含：

- 每卷的**暂定主题**（一句话概括）
- 每卷的**章节数**
- 简要说明本卷在故事中的位置（开场/发展/转折/高潮/收尾）
- 文件输出路径

等待用户明确回复（如"确认"、"开始"、"ok"）后再继续。用户可能会要求调整卷数、章节分配或主题方向——此时更新方案并再次确认。

规划方案示例：

```markdown
计划将《XX》分为 5 卷，章节规划如下：

| 卷 | 主题 | 章节数 | 故事阶段 | 输出文件 |
|----|------|--------|----------|----------|
| 卷一 | 初入江湖 | 60章 | 建置与开端 | `chapters/volume-01/index.md` |
| 卷二 | 风云渐起 | 80章 | 冲突升级 | `chapters/volume-02/index.md` |
| 卷三 | 绝境逢生 | 70章 | 重大转折 | `chapters/volume-03/index.md` |
| 卷四 | 暗流涌动 | 90章 | 高潮铺垫 | `chapters/volume-04/index.md` |
| 卷五 | 天下归心 | 50章 | 终局收尾 | `chapters/volume-05/index.md` |

总览文档：`chapters/index.md`

确认后我开始逐卷规划。
```

### 3. 逐卷生成

按确认的方案，一次生成一卷。每卷使用 `references/volume-template.md` 的结构，填充以下内容：

#### 3.1 基础信息
- 卷号与标题
- 章节范围（第 X 章 – 第 Y 章，共 N 章）
- 本卷核心主题（1–2 句话）

#### 3.2 卷说明
- 本卷在全书中的位置与作用
- 核心冲突或情感主线
- 与前后卷的衔接关系

#### 3.3 出场主要人物
- 列出本卷中有重要戏份的角色
- 标注角色在本卷的状态（登场/成长/转折/退场）
- 如果已有 `presets/characters.md`，从中提取并关联

#### 3.4 阶段划分与章节规划

这是核心部分。将本卷按故事节奏划分为若干阶段（Stage），每个阶段包含多章。模板如下：

```
### 阶段一：<阶段名>（第 X 章 – 第 Y 章，共 N 章）

**阶段目标**：用 1–2 句话说明这个阶段要达到的叙事目的。

| 章节 | 标题 | POV | 主要人物 | 场景 | 主要内容 | 伏笔 / 钩子(可不写) |
|------|------|----|---------|------|----------|------------|
| 1    |      |    |         |      |          |            |
| 2    |      |    |         |      |          |            |
```

**阶段划分建议：**
- 每卷可划分 3–6 个阶段，每个阶段 10–25 章
- 阶段应有清晰的叙事目标：引入冲突、升级对抗、揭示真相、情感转折等
- 相邻阶段之间应有自然的过渡（章节衔接说明）

**章节规划要点：**
- 每章标注核心事件（1 句话抓住要点，方便后续写作时快速理解）
- 如果多视角，标注 POV（Point of View）角色
- 字数估算为可选字段，如果用户有字数目标则填写
- 注意把控节奏：动作场景与文戏交替，高潮与舒缓交替

### 4. 生成总览文档

全部卷规划完成后，生成 `chapters/index.md`，使用 `references/overview-template.md` 的结构，包含：

- **全书章节总览表**：每卷的标题、章节范围、章节数、核心主题、主要人物
- **伏笔清单**：标注每卷的埋设/推进/回收状态
- **全书节奏曲线**：每卷的情感强度或冲突强度概览

### 5. 报告结果

完成后列出生成的文件清单，并提示下一步。

## 节奏与结构原则

规划章节时遵循以下原则：

1. **每卷有独立弧线**：每卷自身是一个小故事，有起承转合，同时推动全书主线。
2. **章章有钩子**：每章的结尾应留下悬念、问题或情感张力，引导读者继续。
3. **高潮分布合理**：避免连续多章高强度导致读者疲劳，也避免长时间平淡。
4. **人物有进有出**：角色登场和退场有安排，避免"突然消失"。
5. **伏笔有始有终**：每个伏笔都标注埋设和计划回收位置，避免遗漏。

## Gotchas

- **不要在没有大纲的情况下直接规划章节**：如果用户完全没有故事框架，建议先用 `novel-design` 完成大纲。
- **不要覆盖已有文件**：`chapters/` 下若已有同名文件，必须先询问用户。
- **不要把正文写进章节规划**：本 skill 只输出章节规划（标题+概要+伏笔），正文由 `novel-write` 处理。
- **章节数不必死守范围**：40–150 是建议值，如果用户指定 20 章或 200 章，按用户的要求来。
- **不要把伏笔清单分散在多处**：伏笔统一在总卷的伏笔清单维护。
- **信息收集未完成前不要开始规划**：必须先确认卷数、每卷章数、特殊要求后再落盘。
- **只生成用户已确认的卷**：如果用户只想先生成一卷看看效果，不要一次生成全部。
