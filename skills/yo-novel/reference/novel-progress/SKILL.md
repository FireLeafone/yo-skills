# novel-progress

增量更新小说项目的 `progress.md`，使其与当前创作状态保持一致，为后续续写提供上下文。

## When to Use

- 用户要求“更新一下进度”、“同步 progress.md”、“记录第 X 章进度”
- 用户完成大纲、人物设定、世界观设定，需要勾选总体进度
- 用户埋下新伏笔或回收旧伏笔
- 用户想让 `progress.md` 反映 `chapters/` 目录里已写章节的最新状态

When NOT to use:

- 用户想从零开始创建小说项目（使用 `novel-create`）
- 用户想写新章节正文（使用 `novel-write`）
- 用户想设计大纲/人物/世界观（使用 `novel-design`）
- 用户只是泛泛地问写作技巧、剧情建议，而不涉及进度文件更新

## Workflow

1. **确定进度文件**
   - 默认使用当前目录下的 `progress.md`。
   - 如果用户明确指定了文件路径，使用该路径。
   - 如果文件不存在，向用户说明并询问是否要从 `novel-create` 的 `progress.md` 模板创建；不要擅自创建。

2. **读取并解析现有进度文件**
   - 读取整个 `progress.md`，识别六个标准区块：`元信息`、`总体进度`、`章节进度`、`人物进度`、`剧情伏笔跟踪`、`时间线进度`。
   - 保留文件原有结构、顺序、自定义标题、备注和未识别的区块。不要重写整个文件。
   - 各区块的字段规则详见 `references/progress-blocks.md`。

3. **收集需要更新的信息**
   - **用户显式提供的信息优先**：章节号、章节名、状态、字数、主要出场人物、情节概况、已完成项、伏笔状态等。
   - **辅助扫描 `chapters/`**：当用户没有明确给出某章信息，但 `chapters/` 目录存在对应章节文件时，可以读取该文件统计字数，并提取已提供的标题；不要凭空编造情节概况或出场人物。
   - **字数统计**：如果项目根目录有 `count_words.py`，直接运行它；否则按「中文字符 + 英文单词数」统计。

4. **生成更新规则并执行**
   - 将准备执行的更新整理成 JSON 规则文件（`updates.json`），包含：
     - `meta_info`：当前章节进度等
     - `overall_progress`：总体进度勾选项
     - `chapters`：章节进度表更新
     - `character_progress`：人物进度勾选项
     - `foreshadowing.add` / `foreshadowing.resolve`：新增/回收伏笔
     - `timeline_progress`：时间线进度勾选项
   - 运行脚本：`python scripts/update_progress.py --input progress.md --spec updates.json`
   - 该脚本会安全地更新标准区块并保留所有未知内容，输出变更摘要 JSON。

5. **落盘前检查（硬性门槛）**

   在确认写回前，逐项核对：

   - [ ] 没有根据章节正文脑补情节概况、出场人物或伏笔
   - [ ] 没有删除或修改用户自定义区块和备注
   - [ ] 没有擅自新增用户未要求的伏笔行
   - [ ] 章节进度表的行数与本次更新目标一致
   - [ ] 复选框和 Markdown 表格语法正确

   任何一项不通过，都先向用户说明，而不是直接写回文件。

6. **返回更新摘要**
   - 用 Markdown 返回简洁的变更摘要，包含：
     - 更新的文件路径
     - 主要改动（新增/修改了哪些章节、伏笔、进度项）
     - 当前总体章节数、最新完成章节
     - 下一步建议（如“可继续写第 X 章”）

## Output Format

返回示例：

```markdown
已更新 `progress.md`：

- 新增章节：第 003 章《雨夜追凶》（已完成，3520 字）
- 更新状态：第 002 章 → 已完成
- 新增伏笔：`神秘来信`（未回收，埋下于第 003 章）
- 回收伏笔：`怀表之谜`（已回收于第 003 章）
- 总体进度：`大纲完成` 已勾选

当前进度：3 / 10 章（30%）。
下一步建议：继续撰写第 004 章。
```

## Common Mistakes

- 在用户没有指定时擅自把进度文件创建成全新模板 → 应先读取现有文件，不存在时询问。
- 重写整个 `progress.md` 导致用户自定义区块或备注丢失 → 增量更新，保留未知内容。
- 根据章节正文“脑补”情节概况、出场人物或伏笔 → 只写用户明确给出或能直接从文件统计的字段。
- 在没有用户确认时覆盖已有的章节摘要或伏笔状态 → 除非用户明确要求，否则保留原值。
- 把章节进度表更新到 `presets/outline.md` 或其他设定文件 → 本技能只负责 `progress.md`。
- 在表格中混合中英文标点或破坏表格对齐 → 写回前检查 Markdown 表格语法。
- 把“写完一章”误判为“写新正文” → 如果只是同步进度，使用本技能；如果用户要求产出正文，转 `novel-write`。

## Resources

- `references/progress-blocks.md`：六个标准区块的字段规范和更新规则
- `scripts/update_progress.py`：安全更新 `progress.md` 的脚本，保留未知区块并输出变更摘要
- 项目根目录的 `count_words.py`（若存在）：用于统计各章节字数
