# 计划自检清单

撰写完完整计划后，按此清单逐项检查。任何未通过的项必须修复后才能交接。

## 检查项

- [ ] **规范覆盖：** 设计文档中的每条需求/章节都有对应任务编号，或已明确标注为排除项。
- [ ] **占位符扫描：** 计划中不存在 TBD、TODO、"稍后实现"、"添加适当的错误处理"、"为上述内容编写测试" 等模糊描述。
- [ ] **类型一致性：** 跨任务使用的类型、方法签名、属性名称保持一致。
- [ ] **需求完整度：** 计划满足设计文档、用户意图和范围边界。
- [ ] **计划遗漏：** 未发现明显遗漏或可执行性缺陷。
- [ ] **外部审阅：** 已使用 [plan-document-reviewer-prompt.md](plan-document-reviewer-prompt.md) 派发审阅子代理，状态为"通过"。
- [ ] **验证策略合理性：** 策略与执行方式匹配，对 `/yo subagent-development` 优先使用 `batched-verification`。

## 修复原则

- 发现问题直接修复，无需重新审阅。
- 发现设计需求没有对应任务，立即补充任务。
- 未通过自检和外部审阅的计划不得交接。
