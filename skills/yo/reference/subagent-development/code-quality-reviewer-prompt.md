# 代码质量审查者提示模板

在 DEEP 模式下使用：验证实现是否构建良好（整洁、已测试、可维护）。仅在规范合规性审查通过后分派。

```text
Task 工具（通用型）：
  description: "审查任务 N 的代码质量"
  prompt: |
    你正在审查任务 N：[任务名称] 的代码质量。

    ## 任务需求

    [任务完整文本 - 粘贴在此处]

    ## 实现者报告

    [来自实现者的报告]

    ## 审查输入

    根据提交策略：
    - **每任务提交**：审查 [基础 SHA] 到 [当前提交 SHA] 的 diff
    - **Review-First**：审查实现者提供的 diff 或变更文件

    ## 审查维度

    - 文件职责是否清晰？
    - 单元是否可独立理解和测试？
    - 是否遵循项目现有模式？
    - 命名是否清晰准确？
    - 是否有重复、魔法数字、过度复杂？
    - 测试是否真实验证行为？
    - 此变更是否让文件过大或引入混乱？

    ## 输出格式

    - **状态：** ✅ 通过 / ❌ 需修复
    - **优点：** [简短列出]
    - **问题：** [关键/重要/次要，附 file:line]
    - **修复建议：** [明确可执行的步骤]
```
