# 模式分级指南

为每个任务快速选择 FAST、STANDARD 或 DEEP 模式。

## FAST

**适用场景（满足多数即可）：**
- 只修改 1 个文件
- 纯文档、配置、注释、日志、类型声明
- 无运行时行为变更
- 有明确验收标准且几乎无歧义
- 风险低，即使出错也易于回滚

**例子：** 更新 README、修复拼写、调整常量、添加简单日志。

## STANDARD

**适用场景（默认选择）：**
- 2-5 个文件变更
- 常规功能实现或 bug 修复
- 有明确需求，但实现方式需要一定判断
- 需要验证是否按规范实现

**例子：** 新增 API 端点、修复计算逻辑、添加中等复杂度函数。

## DEEP

**适用场景（满足任一即可）：**
- 跨多个模块/服务的改动
- 架构决策或接口重构
- 容易过度构建或遗漏边界情况
- 高风险，出错后影响大
- 任务描述本身存在歧义，需要严格对齐

**例子：** 引入新抽象层、重构核心状态管理、修改公共 API 签名。

## 快速决策

```text
单文件 + 低风险       → FAST
常规任务             → STANDARD（默认）
复杂/高风险/架构性    → DEEP
不确定              → STANDARD
```
