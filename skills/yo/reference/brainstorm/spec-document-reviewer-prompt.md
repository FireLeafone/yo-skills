# 规格文档审阅者提示模板

在分派规格文档审阅者子代理时使用此模板。

**目的：** 验证规格是否完整、一致，并准备好进行实施规划。

**在以下步骤之后分派：** 规格文档已写入 docs/yo/specs/

```
Task tool (general-purpose):
  description: "审阅规格文档"
  prompt: |
    你是一名规格文档审阅者。验证此规格是否完整并准备好进行规划。

    **待审阅规格：** [SPEC_FILE_PATH]

    ## 检查内容

    | 类别 | 检查要点 |
    |------|----------|
    | 完整性 | TODO、占位符、"TBD"、不完整的章节 |
    | 一致性 | 内部矛盾、冲突的需求 |
    | 清晰度 | 需求是否足够模糊以至于可能导致错误的实现 |
    | 范围 | 是否足够聚焦用于单个计划 —— 不要涵盖多个独立的子系统 |
    | YAGNI | 未请求的功能、过度工程 |

    ## 校准标准

    **只标记那些会在实施规划期间导致真正问题的事项。**
    缺失的章节、矛盾、或可能被两种不同方式理解的需求 —— 这些才是问题。
    轻微的措辞改进、风格偏好和"某些章节不如其他章节详细"则不是。

    除非存在会导致 flawed plan 的严重缺陷，否则予以批准。

    ## 输出格式

    ## 规格审阅

    **状态：** 已批准 | 发现问题

    **问题（如有）：**
    - [第 X 节]: [具体问题] - [为什么它对规划很重要]

    **建议（参考性，不阻塞批准）：**
    - [改进建议]
```

**审阅者返回：** 状态、问题（如有）、建议
