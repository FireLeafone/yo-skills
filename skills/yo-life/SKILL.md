# Yo-Life 技能索引

Yo-Life 是日常生活辅助技能的集合入口。根据你的需求，自动路由到最合适的子技能。

## 可用子技能

**可以使用 `/yo-life help` 列出下面所有子技能**

| 子技能 | 路径 | 用途 | 典型触发语 |
|--------|------|------|-----------|
| **travel-planner** | `reference/travel-planner/` | 旅行策划与路书交付：简版行程资料 → tripData.json → 交付级 roadbook-v2 HTML/PDF；亦支持目的地推荐、多维信息采集、行程编排与预算编制 | "旅行计划"、"行程规划"、"路书"、"roadbook"、"行程单"、"报价单"、"定制游方案" |

**模式选择提示（travel-planner 两种工作模式）：**
- 有客户简版资料（文本/截图/文档/XLS），要产出给客户看的行程单/路书 → 路书交付模式（默认，自动跑交付流水线）
- 无客户资料，只想「规划去哪玩」「做个旅行计划」 → 旅行策划模式（目的地推荐 → 多维信息采集 → 行程编排 → 预算 → 旅行计划 HTML）

## 路由规则

### 1. 显式指定（优先）

如果用户输入包含 `/yo-life <子技能名>`，直接加载对应子技能：

- `/yo-life travel-planner ...` → 加载 `reference/travel-planner/SKILL.md`

将 `/yo-life <子技能名>` 之后的剩余内容作为任务传递。

### 2. 自动推断

如果用户只输入 `/yo-life` 或 `/yo-life <问题>` 但没有指定子技能名，根据用户意图推断：

| 用户意图关键词 | 匹配子技能 |
|--------------|-----------|
| 旅行、旅游、行程、路书、roadbook、行程单、报价单、定制游、目的地推荐、旅行计划、攻略、去哪玩 | **travel-planner** |

**推断步骤：**
1. 分析用户问题的核心意图
2. 对照上表匹配最合适的子技能
3. 告知用户将使用哪个子技能，然后加载执行

目前本集合仅有 **travel-planner** 一个子技能：意图与旅行/行程相关即直接路由；明显无关时，向用户说明本集合的适用范围，不要强行匹配。

## 工作流程

1. **解析用户输入**：检查是否包含 `/yo-life <子技能名>` 的显式指定
2. **匹配子技能**：
   - 显式指定 → 直接使用对应子技能
   - 未指定 → 根据意图关键词自动推断
3. **加载并执行**：读取对应子技能的 SKILL.md，按其子技能的指令完成任务
4. **结果汇总**：将子技能的执行结果返回给用户

## 注意事项

- 本技能本身不执行具体任务，只负责路由到正确的子技能
- 子技能在 `reference/` 目录下
- 子技能的详细工作流程和规则，请参考各自的 SKILL.md
- **travel-planner** 默认执行交付流水线（正文润色 → 亮点/费用/服务 LLM → 住宿简介 → strict 配图 → 校验 → HTML/PDF），禁止问用户「要不要完整交付」；仅当用户明示「草稿/预览/离线」才用本地占位图，并标注**非交付稿**
- **travel-planner** 交付必须带客户首晚入住/离店日期（`--check-in` / `--check-out`），缺日期时最少字数追问日期，不问「要不要交付」
- **travel-planner** 报价按实查 / 参考（`~`）/ 估算（`≈`）分级标注，不得编造未包含项目、精确票价与班次
